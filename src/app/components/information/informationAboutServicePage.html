Data Models

The Data Models module contains data models of base registers, records and key information solutions of public administration. It allows their structures and metadata to be stored centrally and in a uniform manner. It provides existing and potential users with information on what data is already collected within the framework of public administration and therefore does not need to be redefined. The Data Models module provides:

  • Standardised conceptual and logical data models.
  • A uniform and comprehensive set of metadata, allowing to define the content of databases in a clear and unambiguous way.
  • Data ownership information.
  • Versioning and change control.
  • Relationships between individual databases and their data elements.

Core Data Models are also included in the Data Model Repository. Core Data Models are reusable and extensible data models that capture the fundamental characteristics of entities in a context-neutral way. Public administration bodies can use and extend the unified core data models in the following ways:

  • Development of new information solutions: The Core Data Models can be used as a default starting point for creating conceptual and logical data models of newly developed information solutions.
  • Data exchange between information solutions: Core data models can form the basis of data models for data exchange between existing information solutions.
  • Data integration: Core data models can be used to integrate data coming from different heterogeneous sources.

The repository of data models is closely related to the Terminology and the Code Lists modules.

Podatkovni modeli

Vsebuje podatkovne modele temeljnih registrov, evidenc in ključnih informacijskih rešitev javne uprave. Omogoča, da se njihove strukture in metapodatki hranijo centralno in na enoten način. Obstoječim in potencialnim uporabnikom nudi informacijo kateri podatki se v okviru javne uprave že zbirajo in jih zato ni treba definirati na novo. Modul Podatkovni modeli zagotavlja:

  • Standardizirane konceptualne in logične podatkovne modele,
  • Enoten in širok nabor metapodatkov, ki omogoča jasno in nedvoumno opredelitev vsebine podatkovnih zbirk,
  • Podatke o lastništvu podatkov,
  • Verzioniranje in nadzor nad spremembami,
  • Relacije med posameznimi podatkovnimi zbirkami in njihovimi podatkovnimi elementi.

V Repozitorij podatkovnih modelov so vključeni tudi jedrni podatkovni modeli. Jedrni podatkovni modeli so ponovno uporabljivi in razširljivi podatkovni modeli,
ki zajemajo temeljne značilnosti entitet na kontekstualno nevtralen način. Organi javne uprave lahko uporabljajo in razširijo enotne jedrne podatkovne modele za sledeče namene:

  • Razvoj novih informacijskih rešitev: jedrne podatkovne modele se lahko uporabi kot privzeto izhodišče za oblikovanje konceptualnih in logičnih podatkovnih modelov novih informacijskih rešitev.
  • Izmenjava podatkov med informacijskimi rešitvami: Jedrni podatkovni modeli lahko predstavljajo temelj podatkovnih modelov za izmenjavo podatkov med obstoječimi informacijskimi rešitvami.
  • Podatkovna integracija: Jedrne podatkovne modele se lahko uporablja za integracijo podatkov, ki prihajajo iz različnih heterogenih virov.

Repozitorij podatkovnih modelov se tesno povezuje z moduloma Terminologija in Šifranti.
